## 四、命名规范

### 4.1 数据库命名规范

- 表名：小写字母 + 下划线，如 `user_info`、`product_category`
- 字段名：小写字母 + 下划线，如 `user_name`、`create_time`
- 主键：`id` 或 `表名_id`，如 `user_id`
- 外键：`关联表名_id`，如 `category_id`

### 4.2 Java 类命名规范

- 类名：大驼峰命名，如 `UserInfo`、`ProductController`
- 方法名：小驼峰命名，如 `getUserInfo`、`createOrder`
- 变量名：小驼峰命名，如 `userName`、`orderList`
- 常量名：全大写 + 下划线，如 `MAX_PAGE_SIZE`

### 4.3 前端命名规范

- 组件名：大驼峰命名，如 `ProductList`、`UserCenter`
- 变量名：小驼峰命名，如 `productData`、`userInfo`
- CSS 类名：小写字母 + 短横线，如 `.product-list`、`.user-info`

### 4.4 接口命名规范

- RESTful 风格：
  - `GET /api/products` - 获取商品列表
  - `GET /api/products/{id}` - 获取商品详情
  - `POST /api/orders` - 创建订单
  - `PUT /api/users/{id}` - 更新用户信息
  - `DELETE /api/cart/{id}` - 删除购物车商品

## 五、沟通协商机制

### 5.1 日常沟通

- 建立微信群/QQ 群用于日常沟通
- 每日晚上 22:00 前在群内汇报当日进度
- 遇到问题及时在群内反馈

### 5.2 问题协商

1. **一般问题**：组内成员讨论解决
2. **技术分歧**：查阅资料、对比方案优劣，投票决定
3. **重大分歧**：由组长仲裁，或征求指导老师意见

### 5.3 版本控制

- 使用 Git 进行代码和文档版本管理
- 建立统一代码仓库（GitHub/Gitee）
- 每次修改后及时 commit，并写明 commit message
